# Contract: Internationalization (US2)

A lightweight, in-house i18n layer (no runtime dependency — clarify Q1). Display copy only (no restricted content — IX).

## Module shape (`src/i18n/`)

- **`catalog.ts`** — `catalog: Record<Locale, Record<MessageKey, string>>` for `en` (default, LTR) + `ar` (sample, RTL); `DEFAULT_LOCALE = 'en'`; `RTL_LOCALES = ['ar']`. Keys namespaced by surface (`auth.*`, `runtime.*`, `dashboard.*`, `profile.*`).
- **`LocaleProvider.tsx`** — context provider; reads/persists the active locale via a versioned localStorage key (mirrors `themeStore`); on change sets `document.documentElement.dir` (`rtl`/`ltr`) and `lang`. Mounted once in `App`.
- **`useT.ts`** — `useT()` → `{ t, locale, dir, formatDate, formatNumber, setLocale }`.

## `t()` behavior

```ts
t('auth.signIn')                  // → active-locale string
t('runtime.saved', { n: 3 })      // → interpolates {n}
t('missing.key')                  // → DEFAULT_LOCALE value; never a raw key or blank (FR-PVR-007)
```

- `formatDate(iso)` / `formatNumber(n)` use built-in `Intl` keyed to the active locale (FR-PVR-008).

## Retrofit scope (priority surfaces — clarify Q3)

`features/auth` (sign-in/recovery), `features/runtime` (assessment runtime), Admin dashboard + User dashboard, and Profile & Privacy wrap user-facing copy in `t()`. Other surfaces adopt incrementally later.

## RTL

- Direction is driven by `<html dir>`; logical CSS used where feasible; the few directional inline styles on priority surfaces are mirrored. Inherently directional charts/icons are explicitly exempt.

## Tests (`tests/i18n/`)

- Switching locale updates rendered copy and **persists** across reloads (SC-004).
- Missing key falls back to the default locale (no raw key/blank) (FR-PVR-007).
- RTL locale sets `document.dir === 'rtl'` and the surface still renders its key controls (SC-005).
- `formatDate`/`formatNumber` produce locale-correct output (FR-PVR-008).
